# AI Product Playbook

Use this reference for AI apps, copilots, agents, model wrappers, workflow automation tools, AI SaaS, and AI features inside existing products.

## Competitor Types

- Model provider: foundation model, API, inference platform.
- Wrapper: thin UX over a model with little workflow ownership.
- Workflow-native AI: embedded in a specific job, with domain context and integrations.
- Vertical agent: AI system that completes multi-step work in a domain.
- Incumbent add-on: existing SaaS that adds AI features to defend its base.
- Open-source/community alternative: local model, OSS agent, template, or automation stack.
- Human service substitute: agency, consultant, assistant, freelancer, or internal team.

## AI-Specific Dimensions

Compare:

- Task quality: correctness, completeness, consistency, hallucination control.
- Evaluation loop: benchmarks, human review, feedback capture, regression testing.
- Data advantage: proprietary data, user workflow data, retrieval quality, freshness.
- Workflow depth: one-shot generation vs multi-step execution vs autonomous completion.
- Integration depth: docs, CRM, email, browser, files, databases, calendars, LMS, payments.
- Reliability: latency, uptime, retry behavior, fallback paths, explainability.
- Cost structure: model cost, inference volume, margins, rate limits.
- Safety and compliance: privacy, data retention, audit logs, permissions, guardrails.
- UX: onboarding, prompt burden, editing, review, collaboration, export.
- Distribution: SEO, templates, communities, partnerships, marketplace, enterprise sales.

## Differentiation Wedges

Look for:

- Better domain context, not just a better prompt.
- Evaluation-backed quality claims.
- End-to-end workflow completion instead of chat-only advice.
- Human-in-the-loop review for high-stakes work.
- Lower setup friction through templates, imports, or guided onboarding.
- Trust artifacts: citations, audit trail, confidence labels, exportable reports.
- Data flywheel: each use improves recommendations, retrieval, or personalization.
- Distribution wedge: niche community, professional workflow, marketplace, or integration.

## Red Flags

- "Uses AI" is the whole positioning.
- Competitor advantage depends only on a model that others can access.
- No evaluation method for output quality.
- No plan for user trust, error recovery, or human review.
- Unit economics ignored.
- Product is a feature an incumbent can ship quickly.

## AI Product Recommendation Pattern

End with:

- The real job-to-be-done.
- Whether the product is a model wrapper, workflow tool, or vertical agent.
- The data/workflow/distribution moat it could build.
- The table-stakes AI capabilities it must match.
- The wedge that is hard for competitors to copy.
