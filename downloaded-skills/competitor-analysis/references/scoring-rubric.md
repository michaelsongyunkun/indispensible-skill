# Scoring Rubric

Use this reference when ranking competitors, assigning threat levels, scoring moats, prioritizing opportunities, or building a comparison matrix.

## Scoring Principles

- Use 1-5 scores as relative judgment, not precise measurement.
- Define what each score means before using it.
- Use "unknown" when evidence is missing.
- Explain scores that drive the recommendation.
- Do not average scores blindly; strategic importance depends on the user's decision.

## Core Dimensions

| Dimension | 1 | 3 | 5 |
| --- | --- | --- | --- |
| Threat level | niche or weak substitute | plausible option for some target users | strong direct competitor likely to win deals/users |
| Product maturity | prototype, thin feature set, unreliable | usable workflow with gaps | polished, broad, reliable, integrated |
| ICP overlap | different buyer/user/job | partial overlap | same buyer, user, job, and urgency |
| Pricing pressure | irrelevant or premium-only | some package overlap | direct price anchor or margin pressure |
| Distribution strength | little visible reach | some SEO/community/sales traction | strong channel, brand, partnerships, or sales engine |
| AI/data advantage | no visible advantage | useful AI/data layer but copyable | proprietary data, evaluation loop, or workflow data flywheel |
| Switching cost | easy to leave | some workflow/data lock-in | high lock-in through data, collaboration, integrations, contracts |
| Trust/compliance | weak proof | adequate trust signals | strong credentials, security, proof, or institutional trust |
| Momentum | stale or declining | steady updates | fast launches, hiring, funding, reviews, or community growth |
| Moat | copyable feature | some defensibility | compounding data, network, workflow, brand, or regulatory moat |

## Composite Views

Use only the composite scores that serve the decision:

- Competitive threat: ICP overlap + product maturity + distribution + switching cost + pricing pressure.
- Strategic moat: data/AI advantage + switching cost + trust + distribution + momentum.
- Opportunity attractiveness: customer pain + competitor weakness + ability to differentiate + GTM feasibility + proof needed.
- Copy risk: table-stakes feature value + incumbent ability to ship + low defensibility.

## Opportunity Priority

| Priority | Meaning | Action |
| --- | --- | --- |
| P0 | urgent wedge, high customer pain, competitors weak, feasible proof | build/test immediately |
| P1 | strong opportunity but needs validation or sequencing | add to near-term roadmap |
| P2 | useful but not decisive | monitor or bundle |
| P3 | low leverage, parity trap, or expensive distraction | avoid for now |

## Score Explanation Template

Use compact explanations:

- Threat level: 4/5 because [same ICP], [strong distribution], but [pricing/feature gap].
- Moat: 2/5 because [feature is useful] but [data/workflow advantage appears copyable].
- Opportunity: P1 because [customer pain], [competitors weak], [proof required].

## Common Scoring Mistakes

- Giving every competitor the same score range.
- Treating famous brands as high threat without ICP overlap.
- Penalizing competitors for missing features the target customer does not care about.
- Overweighting product features while ignoring distribution and trust.
- Averaging dimensions when one blocker, such as compliance or switching cost, dominates the decision.
