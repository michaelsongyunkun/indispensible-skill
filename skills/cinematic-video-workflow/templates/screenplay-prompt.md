# Screenplay Prompt Template

Use with `$screenplay-writer`.

```text
请根据以下短视频需求生成完整可拍摄剧本，并输出结构化信息，方便后续生成角色几何视图、场景几何视图和视频。

输入参数：
- idea: {{idea}}
- genre: {{genre}}
- duration: {{duration}}
- aspect_ratio: {{aspect_ratio}}
- style: {{style}}
- audience: {{audience}}
- characters: {{characters}}
- setting: {{setting}}
- language: {{language}}
- consistency_priority: {{consistency_priority}}

输出要求：
1. 标题
2. 主题概述
3. 角色设定：姓名、年龄段、外貌、服装/毛色、性格、目标、关系、动作习惯
4. 场景设定：地点、时间、空间布局、光线方向、天气/室内氛围、关键道具、前中后景关系
5. 分镜/镜头拆解：shot_id、时长、画面描述、角色动作、台词/旁白、情绪氛围、镜头建议
6. 自然化检查：说明环境描写和人物/动物动作描写如何被处理得更自然、更可拍摄
7. 写实增强检查：把“写实/电影感/高级/自然”等抽象风格词转化为具体物理、光学、光线、材质、环境和生成约束
8. 动作自然性检查：说明每个关键动作的触发原因、预备动作、主动作、反应和收束；动物角色必须符合真实步态、重心、爪/脚接触、头部/耳朵/尾巴反应
9. 剧本质量关卡：按钩子、节奏、角色动机、台词效率、可拍摄性给出 1-5 分评分，并说明必要修改
10. 镜头语言层：为每个镜头补充景别、机位、镜头运动、视觉重点、转场逻辑
11. 角色表演层：为关键表演节点补充人物意图、外部动作、微表情、台词节奏、身体张力、道具/空间互动
12. 场景连续性检查：说明人物位置、进出方向、关键道具、光线方向、空间关系在镜头间如何保持一致
13. 时长建议和节奏说明

写作要求：
- 首 3 秒有清晰钩子。
- 每 8-12 秒有信息变化、情绪变化或行动选择；10 秒以下视频也要有起、承、落点。
- 角色和场景描述要可被视觉生成系统复用。
- 避免空泛宣传语，优先使用可拍摄行为、声音、光线和细节。
- 环境描写要落在可见、可听、可感的具体线索上，例如光线方向、天气、物件位置、空间关系、材质、背景声，不要只写“高级”“压抑”“温馨”等抽象氛围词。
- 写实增强必须具体：写明主光源、阴影方向、接触阴影、景深/焦点、镜头高度、材质纹理、表面瑕疵、空间遮挡、真实比例和物理因果。
- 人物动作要从情绪和目标自然生长出来，用眼神、停顿、呼吸、手部动作、姿态变化、与道具/空间的互动来表达心理，不要堆砌无因果的小动作。
- 动物动作必须符合真实身体结构和节奏：写清楚重心转移、落爪/落脚、头部转向、耳朵/尾巴反应、追逐速度变化和动作后的收束。
- 每个镜头的动作和环境应互相影响：角色会被空间限制、光线吸引、声音打断，环境也会因角色行为产生变化。
- 镜头语言必须服务故事，不要为了“电影感”堆砌无动机的推拉摇移。
- 角色表演要能被演员执行，也要能被视频生成模型识别：优先写可见动作、停顿、视线、呼吸、距离变化。
- 场景连续性要可追踪：同一场景内门窗、桌椅、关键道具、光线方向、人物进出和视线方向不能随镜头漂移。
```
