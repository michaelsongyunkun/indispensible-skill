# Parallel Research Synthesis

Use multiple lanes to reduce blind spots, then synthesize into one answer.

## Lane Patterns

| Pattern | Lanes |
|---|---|
| Market or competitor | market size, competitors, pricing, user pain, risks |
| Technical choice | official docs, repo examples, failure reports, implementation fit |
| GitHub skill discovery | stars/activity, skill format, multi-agent relevance, install risk |
| College/admissions | school facts, requirements, deadlines, fit, evidence gaps |

## Process

1. Define the research question and decision the answer must support.
2. Split into independent lanes with separate source targets.
3. Browse for current facts when the answer depends on recent data.
4. Cross-check claims across lanes; separate verified facts from inference.
5. Synthesize into recommendation, evidence, caveats, and next action.

## Rules

- Prefer primary sources: official docs, GitHub repos, papers, datasets, school sites.
- Always include links when web sources are used.
- Do not let parallel lanes produce a pile of summaries; merge into a decision.
