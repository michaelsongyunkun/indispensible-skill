# Resume Rewrite Framework

Use this reference for any actual resume rewrite.

## Evidence Ladder

| Evidence level | What exists in source material | Safe verbs | Risky verbs |
|---|---|---|---|
| Strong | clear ownership, method, scope, and result | led, designed, built, launched, 主导, 搭建, 落地 | none if supported |
| Medium | clear action and deliverable, partial scope | drove, improved, coordinated, 负责, 推动, 优化, 协同 | 主导, 管理, 从0到1 |
| Weak | participation with few details | supported, assisted, contributed, 参与, 支持, 协助 | led, owned, architected, 主导 |
| Missing | no source evidence | ask the user, mark as pending | any concrete claim |

Default to the lower evidence level when unsure.

## STAR/XYZ Bullet Shape

Use one compact sentence whenever possible:

`[Context/Problem] + [Action with method/tool] + [Result with scope or business meaning]`

English pattern:

`Improved [metric/process] by [action/method], resulting in [measurable outcome or deliverable].`

Chinese pattern:

`围绕[业务问题]，通过[具体动作/工具/方法]完成[产出]，带来[结果/影响]。`

If no result is available:

`围绕[业务问题]，通过[具体动作]产出[可验证交付物]，为[后续决策/执行]提供支持。`

## Quantification Rules

Use numbers in this order:

1. Source number: already present in resume or user material.
2. Derived number: arithmetic from source facts, such as conversion rate from users and purchases.
3. Confirm-needed number: plausible but not sourced; mark `[待确认]`.
4. No number: use concrete deliverables instead.

Do not write empty percentages such as "提升 30%" without baseline, sample, or business meaning. Prefer "将处理时长从 5 天缩短至 3 天" over "效率提升 40%" when both values exist.

## Before/After Discipline

For each changed bullet, capture:

- Original: quote exactly or preserve field path.
- Rewrite: final replacement text.
- Why: one sentence tied to hiring value.
- Risk: `none`, `derived`, `needs confirmation`, or `overclaim reduced`.

Do not silently introduce facts. If a rewrite needs a missing fact, add a question:

`[待确认：这个项目覆盖了多少用户/商家/样本？]`

## Strong Resume Language

Prefer observable work:

- analyzed user feedback,
- built dashboard,
- drafted PRD,
- coordinated launch,
- synthesized interviews,
- optimized SQL query,
- delivered weekly report,
- defined evaluation rubric,
- maintained annotation pipeline,
- resolved support tickets.

Avoid unsupported traits:

- 深度理解,
- 熟悉掌握,
- 具备良好沟通能力,
- strong learning ability,
- team player,
- responsible for various tasks.

Replace traits with evidence:

- Bad: `具备较强数据分析能力`
- Better: `使用 SQL 汇总 12 周转化数据，定位注册后首日流失最高的 3 个步骤`

## Chinese Resume Style

Use role-specific nouns, concrete verbs, and compact clauses. Avoid academic self-evaluation unless the resume is for scholarships or research roles.

Good rhythm:

`场景 + 动作 + 产出 + 影响`

Example:

`针对新用户首周留存下降问题，拆分注册、激活、首购 3 段漏斗并输出周报看板，支持运营团队调整触达节奏。`

## English Resume Style

Start with an action verb. Keep one tense convention:

- Current role: present tense for ongoing work.
- Past role: past tense.

Example:

`Analyzed onboarding funnel data across 20K users and identified three drop-off points, informing lifecycle messaging experiments.`

## Compression Rules

When space is limited:

1. Keep JD-relevant achievements.
2. Keep results and tools.
3. Remove internal process details.
4. Merge repetitive bullets.
5. Delete unrelated awards, clubs, or coursework unless they prove a required skill.
