# Continuity Ledger

Use this reference when planning or continuing long-form fiction where facts, promises, powers, items, injuries, and faction states must remain consistent.

## Story Bible Sections

Maintain these ledgers when the project spans multiple chapters:

| Ledger | Track |
| --- | --- |
| Timeline | Chapter number, date/season, location, travel time, deadlines, cooldowns, injuries healing. |
| Character state | Realm, injuries, secrets known, vows, debts, relationships, public reputation, emotional pressure. |
| Faction state | Goals, alliances, grudges, resources, public stance, hidden agenda, recent losses. |
| Power state | Techniques learned, limitations, costs, counters, breakthroughs, failed breakthroughs, backlash. |
| Inventory | Artifacts, pills, talismans, manuals, tokens, money, rare materials, damaged or spent items. |
| Foreshadowing | Plant, chapter, apparent meaning, true meaning, planned payoff, current status. |
| Reader promises | Mysteries, revenge targets, romance tension, rivalries, vows, prophecies, bargains, unresolved hooks. |

## Continuation Workflow

Before continuing supplied text:

1. Extract current chapter facts into the ledgers above.
2. Mark what each present character knows versus what the reader knows.
3. Identify active promises from the last 1-3 scenes.
4. Check power and inventory availability before solving the scene.
5. Continue from the last emotional beat and the current tactical problem.
6. Update the implicit ledger in the output or notes if the user asks for planning support.

## Foreshadowing Rules

Good xianxia foreshadowing often plants:

- A strange rule in the cultivation system.
- A flaw in a technique that later becomes a key.
- An elder's suspicious kindness or silence.
- A resource with a mundane use and a hidden metaphysical meaning.
- A rumor that is wrong in detail but right in direction.
- A debt, oath, token, wound, or witness that returns at the worst time.

Payoffs should feel earned: the reader should be able to remember the plant after seeing the reveal.

## Continuity Audit

Before final output, check:

- A character does not know information they never learned.
- Injuries, cooldowns, poison, tribulation backlash, or damaged artifacts do not vanish without cause.
- Travel and cultivation time fit the timeline.
- Titles and forms of address match current status.
- A faction's response fits its incentives and previous losses.
- A promise is not forgotten after the chapter hook exploited it.
- A power, pill, talisman, or artifact is not available if already spent, broken, sealed, or hidden elsewhere.
